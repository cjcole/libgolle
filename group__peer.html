Detailed Description

This module helps with the maintenance of a set of peers and the distribution of keys. For a more detailed explanation of how the key distribution protocol works, see Key Generation and Distribution.

Enumeration Type Documentation

enum golle_peer_key_state

Identifies what state the key for a peer set is in.

Enumerator:
GOLLE_KEY_UNDEFINED 

The state of the key is undefined.

GOLLE_KEY_EMPTY 

The p, q, and g elements must be set.

GOLLE_KEY_INCOMPLETE 

Not all peers have contributed to the key yet.

GOLLE_KEY_READY 

The key is ready to be used.

Gets the full public key, if ready.

Parameters
setThe set to get the key for.
Returns
If set is not NULL, and the state of the key is GOLLE_KEY_READY, then return the key. Otherwise, return NULL.
Note
The key returned is a pointer to the key held internally by the state. Do not free it, do not alter it. When the state is deleted, the key becomes invalid.
GOLLE_EXTERN golle_peer_key_state golle_peers_get_state ( golle_peer_set_t *  set)

Get the current state of the key.

Parameters
setThe set to check.
Returns
The state of the key. If set is NULL, the key state is GOLLE_KEY_UNDEFINED. If the key has not been set up via golle_peers_set_key(), the state is GOLLE_KEY_EMPTY. If not all peers have contributed to the global key via golle_peers_commit() and golle_peers_verify(), the key state is GOLLE_KEY_INCOMPLETE. If all peers have contributed to the key, then the key is ready to use and the state is GOLLE_KEY_READY.

GOLLE_EXTERN golle_error golle_peers_set_key ( golle_peer_set_t *  set,
golle_key_t *  key 
)

Set the public parts (not including h) of the key.

Parameters
setThe peer set to set the key for.
keyThe key, with the appropriate members set, to use.
Returns
GOLLE_OK if successful. GOLLE_ERROR if set is NULL. GOLLE_EINVALID if any part of the key is invalid. GOLLE_EMEM if a memory allocation error occured.
Note
Setting a NULL key will result in the key being emptied and the state set back to GOLLE_KEY_EMPTY.
If a new key is set successfully, the key state will either be GOLLE_KEY_READY if no peers are added, or GOLLE_KEY_INCOMPLETE if there are peers in the set.

GOLLE_EXTERN golle_error golle_peers_verify ( golle_peer_set_t *  set,
golle_peer_t  peer,
golle_bin_t *  rkeep,
golle_bin_t *  secret 
)

Verify a commitment to a key for a peer.

Parameters
setThe set to verify with.
peerThe peer to verify for.
rkeepThe rkeep of the commitment record.
secretThe secret of the commitment record.
Returns
GOLLE_OK if the commitment was successfully verified. GOLLE_ERROR if any parameter is NULL. GOLLE_EINVALID if the peer is not a member of the set. GOLLE_EEXISTS if a key for the peer has already been verified. GOLLE_ENOTFOUND if a commitment for the peer does not exist. GOLLE_ENOCOMMIT if the commitment test fails.
Note
If verification is successful, the peer's key will be accumulated into the global public key.
If the function results in all peers having verified their key, the state of the peer set will become GOLLE_KEY_READY.
